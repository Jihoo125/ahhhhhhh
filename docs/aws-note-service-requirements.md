# 온라인 메모/노트 서비스 요구사항 명세서 (AWS 기반)

## 1. 개요

### 1.1 목적
본 문서는 AWS 클라우드 환경을 기반으로 구축되는 온라인 메모/노트 서비스의 기능 요구사항과 비기능 요구사항을 정의한다. 본 서비스는 개인 사용자와 팀 사용자가 웹 및 모바일 환경에서 메모를 생성, 수정, 삭제, 검색, 공유할 수 있도록 지원하며, 데이터는 안전하고 확장 가능한 AWS 인프라에 저장된다.

### 1.2 범위
서비스는 다음 기능을 제공한다.
- 사용자 인증 및 계정 관리
- 메모 생성, 수정, 삭제, 복구
- 폴더, 태그, 즐겨찾기, 핀 고정 기반 정리
- 제목/본문/태그 검색
- 링크 및 사용자 초대를 통한 공유
- 다중 디바이스 간 동기화
- 선택적 알림 기능
- 운영 모니터링 및 향후 확장 구조

### 1.3 주요 사용자
- **일반 사용자**: 개인 메모 작성 및 관리
- **팀 사용자**: 메모 공유 및 협업
- **관리자**: 운영, 모니터링, 정책 관리

---

## 2. 시스템 아키텍처 (AWS 기반)

### 2.1 전체 구성
- **프론트엔드**: React / Next.js
- **백엔드**: AWS Lambda + Amazon API Gateway
- **인증**: Amazon Cognito
- **데이터 저장소**
  - 메타데이터: Amazon DynamoDB
  - 파일/이미지: Amazon S3
- **검색**: Amazon OpenSearch Service (선택)
- **캐싱**: Amazon ElastiCache for Redis
- **배포**: Amazon CloudFront + S3 정적 호스팅
- **모니터링**: Amazon CloudWatch, AWS X-Ray, Amazon SNS

### 2.2 아키텍처 흐름
```text
사용자 → CloudFront → API Gateway → Lambda → DynamoDB / S3 / OpenSearch
```

### 2.3 아키텍처 원칙
- 서버리스 중심 설계로 운영 부담 최소화
- 인증, 데이터, 파일 저장, 검색을 AWS 관리형 서비스로 분리
- API 계층과 저장소 계층 간 느슨한 결합 유지
- 향후 협업, AI, 음성 기능 확장을 고려한 구조 채택

## 3. 기능 요구사항

### 3.1 사용자 인증
#### 3.1.1 기본 기능
- 회원가입
- 로그인
- 로그아웃
- 비밀번호 재설정
- 이메일 인증

#### 3.1.2 고급 인증
- Google, Apple 등 소셜 로그인 지원
- Amazon Cognito 기반 JWT 인증 적용
- 액세스 토큰 만료 및 재발급 처리

#### 3.1.3 세부 요구사항
- 이메일 인증 완료 후에만 서비스 주요 기능 접근 가능해야 한다.
- 비밀번호 재설정 메일 발송 기능을 제공해야 한다.
- 인증 실패 시 명확한 오류 메시지를 제공해야 한다.
- 토큰 위변조 방지 및 만료 검증이 필수다.

### 3.2 메모 관리
#### 3.2.1 메모 생성
- 텍스트 메모 생성
- 이미지 첨부 지원
- 체크리스트 형식 지원
- 자동 저장(Auto-save) 지원

#### 3.2.2 메모 수정
- 실시간 수정 반영
- 마지막 수정 시각 저장
- 선택적으로 버전 히스토리 저장 가능

#### 3.2.3 메모 삭제
- Soft delete 방식의 휴지통 기능 지원
- 휴지통에서 복구 가능해야 함
- 영구 삭제 기능 제공

### 3.3 메모 정리 기능
- 폴더 생성 및 폴더별 분류
- 태그 추가 및 제거
- 즐겨찾기 설정
- 핀 고정으로 상단 노출

### 3.4 검색 기능
- 제목 검색
- 본문 검색
- 태그 기반 검색
- Full-text search 지원 (OpenSearch 연동 시)

### 3.5 공유 기능
- 링크 기반 공유
- 이메일 기반 사용자 초대
- 권한 설정
  - 읽기 전용
  - 편집 가능

### 3.6 동기화
- 여러 디바이스 간 실시간 동기화
- 오프라인 작성 후 온라인 복귀 시 동기화
- 충돌 발생 시 최신 수정본 우선 또는 병합 정책 정의 필요

### 3.7 알림 기능 (선택)
- 메모 공유 시 알림
- 메모 수정 시 알림
- 추후 푸시 알림 확장 가능

## 4. 비기능 요구사항

### 4.1 성능
- API 평균 응답 시간은 **300ms 이하**를 목표로 한다.
- 동시 사용자 **10,000명 이상**을 처리할 수 있어야 한다.
- 검색 기능은 일반 조회 API와 분리하여 확장 가능해야 한다.

### 4.2 확장성
- Lambda 기반 서버리스 구조를 채택한다.
- 요청 증가에 따라 자동 확장이 가능해야 한다.
- 데이터 저장소는 트래픽 증가에 대응할 수 있어야 한다.

### 4.3 가용성
- 서비스 가용성 목표는 **99.9% 이상**이다.
- Multi-AZ 기반 관리형 서비스를 우선 사용한다.
- 장애 발생 시 빠른 복구가 가능해야 한다.

### 4.4 보안
- 전 구간 HTTPS 적용
- IAM 기반 최소 권한 원칙 적용
- 저장 데이터 암호화
  - S3: AES-256 또는 KMS 기반 암호화
  - DynamoDB: AWS KMS 기반 암호화
- 민감 정보는 로그에 평문으로 저장하지 않아야 한다.

### 4.5 데이터 무결성
- 필요한 경우 트랜잭션 처리 지원
- 정기 백업 및 복구 절차 제공
- 삭제/복구/동기화 과정에서 데이터 손실을 최소화해야 한다.

## 5. 데이터 모델 (예시)

### 5.1 User 테이블 (DynamoDB)
| 필드 | 타입 | 설명 |
|---|---|---|
| userId | PK | 사용자 ID |
| email | string | 이메일 |
| createdAt | timestamp | 생성일 |

### 5.2 Note 테이블
| 필드 | 타입 | 설명 |
|---|---|---|
| noteId | PK | 노트 ID |
| userId | GSI | 사용자 ID |
| title | string | 제목 |
| content | string | 내용 |
| tags | list | 태그 목록 |
| createdAt | timestamp | 생성일 |
| updatedAt | timestamp | 수정일 |
| deletedAt | timestamp/null | 삭제일 (휴지통용) |
| isPinned | boolean | 핀 고정 여부 |
| isFavorite | boolean | 즐겨찾기 여부 |

### 5.3 선택적 확장 엔티티
- **Folder**: 폴더 정보 및 계층 구조
- **SharePermission**: 공유 대상 및 권한 정보
- **VersionHistory**: 메모 버전 이력 저장
- **Attachment**: 이미지/파일 메타데이터

## 6. API 설계 (예시)

### 6.1 메모 생성
- `POST /notes`

### 6.2 메모 조회
- `GET /notes/{id}`

### 6.3 메모 목록 조회
- `GET /notes`

### 6.4 메모 수정
- `PUT /notes/{id}`

### 6.5 메모 삭제
- `DELETE /notes/{id}`

### 6.6 추가 고려 API
- `POST /auth/signup`
- `POST /auth/login`
- `POST /notes/{id}/share`

- `POST /notes/{id}/restore`

## 7. AWS 서비스 매핑

| 기능 | AWS 서비스 |
|---|---|
| 인증 | Amazon Cognito |
| API | Amazon API Gateway |
| 서버 실행 | AWS Lambda |
| DB | Amazon DynamoDB |
| 파일 저장 | Amazon S3 |
| 검색 | Amazon OpenSearch Service |
| CDN | Amazon CloudFront |
| 로그 및 모니터링 | Amazon CloudWatch |
| 트레이싱 | AWS X-Ray |
| 알림 | Amazon SNS |
| 캐싱 | Amazon ElastiCache for Redis |

## 8. 운영 및 모니터링
- CloudWatch Logs를 통한 애플리케이션 및 인프라 로그 수집
- CloudWatch Alarm을 통한 오류율, 지연 시간, 사용량 모니터링
- AWS X-Ray를 통한 요청 추적 및 병목 분석
- SNS를 통한 장애 및 에러 알림 전파

---

## 9. 확장 고려사항
- **협업 실시간 편집**: WebSocket 또는 AWS AppSync 기반 동시 편집
- **AI 요약 기능**: Amazon Bedrock 연계를 통한 메모 요약/추천
- **음성 메모**: Amazon Transcribe를 이용한 음성 텍스트 변환
